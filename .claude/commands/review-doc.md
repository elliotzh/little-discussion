请审阅以下文档。审阅前，先完成以下准备工作：

1. 查找文档同目录下是否存在 `.meta.json` 文件（如 `plan.md` 对应 `plan.meta.json`）。如果存在，读取其中的目标受众、使用方式、语气要求和约束条件，并将这些信息作为审阅的上下文贯穿所有维度。
2. 读取 CLAUDE.md，重点关注：术语一致性规则、信息隔离规则、文档体系树、跨文档引用链。
3. 查找 `docs-todo/` 下对应位置是否存在上一次的审阅建议（路径映射：`docs/X/Y.md` → `docs-todo/X/Y.md`）。如果存在，读取作为参考——关注上次提出的问题是否已被修复，以及是否有新问题产生。

## 审阅维度

### 1. 论证逻辑
- 核心论点是否清晰？
- 论证链条是否完整、自洽？
- 是否存在逻辑跳跃或隐含假设未被说明？

### 2. 论据充分性
- 关键论断是否有足够支撑？
- 是否存在需要补充数据、案例或引用的地方？
- 反面观点是否被充分考虑？

### 3. 结构与层次
- 章节划分是否合理？
- 段落间的过渡是否自然？
- 整体叙事节奏是否恰当？

### 4. 受众适配性（基于 .meta.json）
- 内容深度和表达方式是否匹配目标受众的认知水平和关注点？
- 是否存在对目标受众而言过于技术化或过于浅显的内容？
- 整体语气是否符合使用场景（如 PPT 演示需简洁有力，白皮书需详实严谨）？

### 5. 语言质量
- 表达是否准确、流畅？
- 是否存在冗余或模糊的表述？
- 术语使用是否一致？

### 6. 术语一致性与信息隔离（基于 CLAUDE.md）
- 术语是否符合 CLAUDE.md 中的术语表？
- 如审阅 `service-brief-enterprise.md` 或 `service-brief-smb.md`，是否泄露了应留在内部文档中的敏感信息？

### 7. 文档体系一致性（基于 CLAUDE.md）
- 当前文档是否已在 CLAUDE.md 的「文档体系」目录树中注册？如果是新文件，提示需要补充。
- CLAUDE.md「跨文档引用」中涉及当前文档的行号是否仍然准确？逐条验证，报告漂移的行号。
- 当前文档内部引用的其他文档路径是否存在于磁盘上？

### 8. 上次建议的跟进（基于 docs-todo/ 中的已有建议）
- 上次提出的问题中，哪些已被修复？
- 哪些仍然存在？
- 是否有新产生的问题？

## 更新审阅建议文件

审阅完成并向用户展示结果后，将**当前仍存在的待改进项**写入 `docs-todo/` 下对应位置的文件（路径映射：`docs/X/Y.md` → `docs-todo/X/Y.md`）。

这是一次**完整覆写**而非追加——文件内容应反映文档当前状态的所有待改进项。已修复的问题不再保留。

文件格式：

```markdown
# [文件名] 审阅建议

> 最近更新：[日期]

### [建议标题]（严重程度：高/中等/轻微）
- 问题描述
- 建议修改方向
```

如果审阅后没有待改进项，**删除** `docs-todo/` 下对应的建议文件（如果存在）。不保留空文件或"无建议"占位内容——文件不存在即代表当前无待处理建议。这也确保下一次审阅时不会暴露"曾经完成过一轮修正"的历史信息。
